# Playwrong V2 Plan

## Goals

- Make LLM operation paths deterministic and low-exploration.
- Improve runtime reliability under navigation jitter and plugin misses.
- Standardize plugin capability contracts and failure recovery.

## V2 Priority Tracks

1. Connection and runtime status (P0)
- Extension popup must show server/bridge status explicitly.
- Offline state must not crash plugin management actions.
- Endpoint changes in popup must propagate to extension websocket runtime.

2. Action receipt and recovery contract (P0)
- `apply/call` should return structured receipts:
  - before/after URL
  - updated node ids
  - recommended next action on failure

3. Plugin capability contract (P1)
- Unified naming for key semantic ids and page calls.
- Required debug page function per plugin.
- Required Failure Modes section in plugin skill.

4. Test templates for plugin packs (P1)
- Required validation tests:
  - install/enable/disable
  - extract + key node ids
  - one setValue, one call, one post-action assertion

5. Task-level abstraction functions (P2)
- Introduce higher-level actions for common flows:
  - `repo.create`
  - `search.run`
  - `search.nextPage`

## Completed In This Iteration

- Popup connection state card and manual check button.
- Popup auto polling of connection status.
- Plugin actions disabled when server is offline.
- Better offline error messages (instead of hard failure stack traces).
- Background websocket now respects saved `serverWsUrl` from popup config.

## Acceptance Criteria

- With server stopped, popup shows `Server Offline`, no uncaught runtime errors.
- With server started but extension disconnected, popup shows warning state.
- After setting a new endpoint, background reconnects using saved websocket URL.
- Plugin list operations work only when connection is healthy and provide actionable errors otherwise.
